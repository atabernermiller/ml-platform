ml-platform

A unified library for serving, monitoring, tracking, and deploying ML models on AWS — from experiment to production in one import.

pip install ml-platform[all]

01 — Overview

Five pillars, one platform

Config

One ServiceConfig dataclass centralizes all settings — S3, MLflow, context store, and region — loaded from env vars or code.

Serving

Stateful (FastAPI) and stateless (BentoML) serving paths with automatic health checks, request routing, and background tasks.

Monitoring

MetricsEmitter writes CloudWatch EMF to stdout and optionally pushes via PutMetricData. Dashboard generators for Grafana and CloudWatch.

Tracking

ExperimentTracker wraps MLflow for logging params, metrics, and artifacts across training and serving runs.

Infrastructure as Code

AWS CDK constructs for VPC networking, ECS Fargate services with ALB, SageMaker endpoints, self-hosted MLflow (RDS + S3), and CloudWatch alarms with SNS notifications.

State & Context

S3StateManager for periodic model checkpoint upload/restore. ContextStore (DynamoDB or in-memory) for predict→feedback correlation on delayed outcomes.

Serving

Stateful path: Subclass StatefulServiceBase, implement predict() and process_feedback(). The factory builds a full FastAPI app with S3 checkpoints, periodic metrics, and lifecycle hooks.

Stateless path: Wrap any BentoML service with with_platform_monitoring() for the same observability guarantees.

Monitoring

MetricsEmitter emits CloudWatch Embedded Metric Format (EMF) JSON to stdout — zero-config when running in ECS. Supports direct PutMetricData for low-latency alarm signals. Dashboard generators produce ready-to-import JSON for Grafana and CloudWatch.

Tracking

ExperimentTracker manages a single MLflow run per service instance. Automatically logs parameters at startup and periodically records metrics snapshots. Artifacts (model checkpoints, configs) can be logged on demand.

03 — Data Flow

Stateful service lifecycle

Startup sequence

1. Config parsed → 2. S3StateManager restores latest checkpoint → 3. load_state() hydrates model → 4. ExperimentTracker starts MLflow run → 5. MetricsEmitter initialized → 6. on_startup() hook → 7. Background loops begin.

Request cycle

Predict: payload → service.predict() → PredictionResult → metrics emitted as EMF event.
Feedback: request_id + outcome → process_feedback() → optionally retrieves context from ContextStore.
Shutdown: final save_state() → S3 upload.

04 — Serving Paths

Two ways to serve

Stateful — FastAPI

For models that maintain state between requests: bandits, recommenders, online learners. The framework manages the complete lifecycle.

class MyBandit(StatefulServiceBase): def predict(self, payload): action = self.model.select(payload) return PredictionResult( prediction=action, metadata={"confidence": 0.87} ) app = create_stateful_app(MyBandit, config)

Routes: /predict /feedback /health /metrics

Stateless — BentoML

For pure inference services. Wrap any existing BentoML service with platform monitoring — same metrics guarantees, no state management.

import bentoml class MyClassifier(bentoml.Service): def __init__(self): self.monitor = with_platform_monitoring( self, config ) def predict(self, input): result = self.model(input) self.monitor.record_prediction(result) return result

Uses PlatformMonitor → MetricsEmitter + optional ExperimentTracker

Stateful services auto-manage: S3 checkpoints, lifecycle hooks, background metric emission
Stateless services get: EMF metrics, MLflow experiment tracking, dashboard compatibility

05 — Infrastructure

CDK constructs for AWS

1

NetworkConstruct

Provisions a VPC with public/private subnets, NAT gateways, and security groups.

2

EcsServiceConstruct

Fargate task definitions, auto-scaling, ALB with health checks on /health.

3

MLflowConstruct

Self-hosted MLflow: Aurora RDS backend, S3 artifact store, Fargate server.

4

MonitoringConstruct

CloudWatch alarms on reward, cost, latency P95, exploration ratio. SNS email alerts.

5

SageMaker Endpoint

Alternative compute: SageMaker real-time endpoints with auto-scaling policies.

Everything is wired

All constructs share the MLPlatform CloudWatch namespace with service dimension — metrics emitted by MetricsEmitter automatically appear in MonitoringConstruct alarms and generated dashboards. No glue code needed.

06 — Quick Start

From zero to production

1

Configure

Define ServiceConfig or set ML_PLATFORM_* env vars for S3, MLflow URI, context store backend.

2

Implement

Subclass StatefulServiceBase or use with_platform_monitoring() for BentoML.

3

Deploy

CDK stack with NetworkConstruct → EcsServiceConstruct → MonitoringConstruct.

4

Observe

CloudWatch metrics stream automatically. Import generated Grafana dashboards. MLflow UI for experiments.

Full Example from ml_platform import ServiceConfig from ml_platform.serving import ( StatefulServiceBase, create_stateful_app, PredictionResult ) config = ServiceConfig( service_name="reward-bandit", aws_region="us-east-1", s3_checkpoint_bucket="my-checkpoints", mlflow_tracking_uri="http://mlflow.internal:5000", ) class RewardBandit(StatefulServiceBase): def predict(self, payload: dict) -> PredictionResult: action = self.model.select_arm(payload["context"]) return PredictionResult(prediction=action, metadata={"arm": action}) def process_feedback(self, request_id: str, feedback: dict): self.model.update(feedback["reward"]) def save_state(self, path: str): self.model.save(path) def load_state(self, path: str): self.model.load(path) def metrics_snapshot(self) -> dict: return {"cumulative_reward": self.model.total_reward} app = create_stateful_app(RewardBandit, config) # uvicorn my_service:app --host 0.0.0.0 --port 8080
